# Hoppe — Adobe Animate 아트 작업 가이드 (HTML5 Canvas / CreateJS)

이 문서는 **애니메이터**를 위한 것입니다. 아래 규칙만 지켜서 작업·핸드오프하면,
개발자가 Electron 앱에 그대로 붙입니다. (JS 연동은 개발자 몫 — 아티스트는 **문서 타입 /
구조 / 라벨 / 투명 / 발 기준**만 신경 쓰면 됩니다.)

---

## 0. 큰 그림

Animate의 **HTML5 Canvas 문서**로 작업 → 퍼블리시하면 **CreateJS** 형식(`.js` 라이브러리 +
이미지)으로 나옵니다. 벡터 모션 트윈이 런타임에 그대로 재생되어 "플래시 느낌"이 유지됩니다.
앱은 캐릭터를 **300×300 박스** 안에서 상태별로 재생합니다.

---

## 1. 문서 설정 (필수)

- **File ▸ New ▸ HTML5 Canvas** 로 새 문서. (ActionScript / AIR 문서 아님! 기존 FLA면 문서 타입을 HTML5 Canvas로 변환)
- **스테이지 크기: 300 × 300 px**
- **배경: 투명 유지** — 배경색을 칠하거나 불투명 사각형을 깔지 말 것.
- **FPS: 24** 권장.
- **방향: 오른쪽을 보는 모습만** 제작. 왼쪽은 코드가 자동 좌우반전.
- **발 기준: 캐릭터의 발바닥이 스테이지 하단 중앙**에 오도록 배치. (바닥/창 모서리에 서는 기준점)

---

## 2. 구조 — 하나의 심볼 + 프레임 라벨 (권장)

1. **MovieClip 심볼 하나**를 만들고, **Linkage(연결) 이름 = `Hoppe`** 로 export 설정
   (심볼 우클릭 ▸ Properties ▸ **Export for runtime**, Class/이름 `Hoppe`). 그 심볼을 스테이지에 1개 배치.
2. 그 심볼 **내부 타임라인**에 상태별 구간을 만들고, 각 구간 시작 프레임에 **프레임 라벨**을 답니다.
   라벨은 **정확히 소문자로 아래 6개** (코드와 1:1, 오타 금지):

   | 라벨 | 상태 | 재생 |
   |---|---|---|
   | `idle` | 대기 | 루프 |
   | `walk` | 걷기 | 루프 |
   | `drag` | 잡혔을 때 | 루프 |
   | `fall` | 낙하 | 루프 |
   | `land` | 착지(짧게) | **1회(원샷)** |
   | `sleep` | 수면 | 루프 |

3. 각 구간은 그 상태의 애니메이션. 상태 전환은 앱이 `gotoAndPlay("<라벨>")`로 처리하므로,
   **구간끼리 자연스럽게 이어지도록** 만들면 됩니다(루프 구간은 끝→시작이 매끄럽게).

> 대안(비권장): 상태별 개별 심볼(`clip_idle`, `clip_walk`…)로 export. 관리 포인트가 늘어서 위 방식 권장.

---

## 3. 애니메이션 제작 규칙

- **모션 트윈 / 클래식 트윈 적극 사용** — 플래시 느낌의 핵심. 빠른 동작 등 꼭 필요한 곳만 프레임 바이 프레임.
- **벡터 우선**(스케일·부드러움 유리). 단 아주 복잡한 벡터는 런타임 부하 → 그 부분만 비트맵/스프라이트시트로.
- **부위 분리**: 팔·머리·눈처럼 따로 움직일 부분은 별도 심볼/레이어(중첩 MovieClip)로. Animate가 CreateJS 컨테이너로 내보냅니다.
- **기준점(Transform point)** 을 일관되게(예: 발밑) — 회전·스케일 축이 흔들리지 않게.

---

## 4. 피해야 할 것 (HTML5 Canvas 미지원/주의)

- **ActionScript 코드 금지** — HTML5 Canvas 문서는 JS입니다. 타임라인 스크립트는 최소화(로직은 앱이 담당).
- 일부 **필터 / 블렌드 모드 / 고급 마스크**는 CreateJS에서 미지원·부정확 → 사용 자제하거나 해당 부분만 래스터화.
- **캐릭터 안에 텍스트(대사) 넣지 말 것** — 대사는 앱의 말풍선이 담당.
- **투명 배경 유지** — 불투명 배경 금지.
- 폰트/이미지 임베드 누락 주의.

---

## 5. 퍼블리시 & 핸드오프

**File ▸ Publish Settings (HTML5 Canvas)**
- JavaScript 라이브러리(`.js`) + `images/` 출력.
- 비트맵을 쓰면 **"Export as spritesheet"(텍스처 아틀라스)** 옵션 켜기(파일 수·성능 개선).
- CreateJS는 앱에서 로컬 번들하므로 CDN/hosted 옵션은 신경 안 써도 됨.

**레포 `assets/animate/` 에 아래를 넣어 전달:**
1. **`.fla` 원본** (수정용, 필수)
2. 퍼블리시 산출물: **`<이름>.js`(라이브러리)** + **`images/`**(또는 아틀라스 png+json)
3. 사용한 폰트·소스 에셋(있으면)
4. 라이브러리 심볼 export 이름(위 규칙대로면 `Hoppe`)과 산출 `.js` 파일명 메모

---

## 6. 핸드오프 전 체크리스트

- [ ] HTML5 Canvas 문서 · 300×300 · **투명 배경**
- [ ] **오른쪽 방향** 기준, **발바닥 = 하단 중앙** 정렬
- [ ] 심볼 Linkage export 이름 `Hoppe`
- [ ] 프레임 라벨 6개: `idle` `walk` `drag` `fall` `land` `sleep` (정확한 소문자)
- [ ] `land`는 원샷, 나머지는 루프(끝→시작 매끄럽게)
- [ ] 필터/블렌드/ActionScript 미사용(또는 래스터화)
- [ ] 캐릭터 내부 텍스트 없음
- [ ] `.fla` + 퍼블리시(`.js` + `images/`) + 폰트 동봉

---

## 7. 참고 — 앱 연동(개발자 몫, 아티스트는 안 해도 됨)

Electron 렌더러가 `createjs.min.js` + 퍼블리시된 `.js`를 로드 → `new lib.Hoppe()` 인스턴스를
캔버스에 그림 → 상태머신이 mode가 바뀔 때 `gotoAndPlay("<라벨>")` 호출. 즉 위 라벨·구조·투명·발
기준만 지키면 나머지는 코드가 붙입니다.

> 통합 검증은 **아주 작은 테스트 export 1개**(심볼 `Hoppe`에 라벨 2개 정도, 단순 도형)만 먼저 주면,
> 개발자가 로더를 붙여 동작을 확인한 뒤 본 작업물을 받는 순서로 진행합니다.
