# 대사 풀 (dialogue.json)

말풍선에 무작위로 노출되는 대사 데이터입니다. 앱 시작 시 로드됩니다
(수정 후 `npm start`로 재실행).

## 구조

카테고리별 문자열 배열입니다. 카테고리는 펫의 상태(mode)와 1:1로 대응하며,
시작 인사용 `greeting`이 추가됩니다.

| 카테고리 | 언제 노출 | 방식 |
|---|---|---|
| `greeting` | 앱 시작 직후 | 이벤트(1회) |
| `idle` | 가만히 있을 때 | 앰비언트(주기적 랜덤) |
| `walk` | 이동 중 | 앰비언트 |
| `sleep` | 잘 때 | 앰비언트 |
| `drag` | 잡혔을 때 | 이벤트(즉시) |
| `fall` | 낙하 중 | (예약: 필요 시 이벤트 연결) |
| `land` | 착지 순간 | 이벤트(확률적) |

### 속마음(inner-thought) 태그

대사는 **문자열** 또는 **`{ "text": "...", "thought": true }`** 객체로 쓸 수 있습니다.
`thought: true`가 붙은 대사는 **그 카테고리(모션)에 속하면서 동시에 속마음**으로 분류되어,
평소 흐름대로 뽑혔을 때 **속마음 말풍선**(어두운 보라 그라데이션 + 생각풍선 꼬리)으로
렌더됩니다. 별도의 카테고리나 타이머가 아니라 해당 모션 대사 풀 안에서 함께 섞여 나옵니다.

```jsonc
"idle": [
  "심심해…",                                  // 일반 말풍선
  { "text": "(사실… 조금 외로웠어)", "thought": true }  // 같은 idle 이지만 속마음 스킨
]
```

속마음 느낌을 살리려면 괄호 `( )`로 감싸는 걸 추천합니다. `thought`는 어느 카테고리
(greeting/idle/walk/sleep/drag/fall/land)에서든 개별 대사에 붙일 수 있습니다.

- **앰비언트**: 8~18초 간격의 무작위 타이머로, 현재 상태 카테고리에서 한 줄 노출
- **이벤트**: 상태 전이 시 즉시 노출(진행 중인 말풍선을 대체)
- **셔플백 방식**: 한 카테고리의 모든 대사를 한 번씩 소진한 뒤 다시 섞음 → 랜덤이지만
  같은 대사가 뭉치지 않고 골고루 나옵니다. 쿨다운(약 4.5초) + 직전 대사 반복 방지 유지.

## 트릭컬 원작 반영

현재 문장은 **오리지널 톤의 임시 대사**입니다(원작 대사 복제 아님). 캐릭터가 정해지면
그 캐릭터의 말투·성격에 맞춰 **직접 쓴 오리지널 대사**로 교체합니다(원작 인게임 텍스트를
그대로 복제하지 않고 스타일만 반영). 캐릭터가 여러 명이 되면 `dialogue.json`을
캐릭터별로 분리하는 구조로 확장합니다.
